# Domain: User

## Mô tả
Aggregate Root quản lý identity data của user trong hệ thống. Source of truth cho thông tin user — các service khác lấy data qua API, không truy cập trực tiếp DB.

---

## Trách nhiệm
- Quản lý thông tin identity của user (profile, credentials, roles, social connections).
- Quản lý role assignments — cả flat (admin-level) lẫn contextual (per-scope/per-org).
- Kiểm soát vòng đời tài khoản (status transitions).
- Phát sinh domain events khi có thay đổi quan trọng.

## Không thuộc trách nhiệm
- Không xác thực credentials — đó là trách nhiệm của oauth2 service.
- Không track failed login attempts — oauth2 service đọc `LoginActivity` và gọi API lock khi cần.
- Không quản lý session — đó là trách nhiệm của oauth2 service.

---

## Cấu trúc Aggregate

```
User
├── UserId                    (Value Object — typed UUID)
├── Email                     (String — validated, unique, immutable)
├── Username                  (String — unique, immutable sau khi set)
├── PhoneNumber               (String — optional, unique)
├── UserPassword              (Value Object — hashed, null nếu social-only)
├── FullName                  (String — optional)
├── UserStatus                (enum: PENDING, ACTIVE, LOCKED)
├── usernameChanged           (boolean — false nếu vẫn đang dùng auto-generated)
├── Set<SocialConnection>     (Entity — owned by User)
├── Set<RoleId>               (Value Object — flat role assignment, no scope context)
├── Set<RoleContext>          (Entity — contextual role assignments per scope+org)
├── createdAt                 (Instant — immutable)
└── updatedAt                 (Instant — cập nhật mỗi khi aggregate thay đổi)
```

> **Design note — roleIds vs roleContexts**: `roleIds` là flat assignment không có scope/org context, được giữ lại cho backward compatibility với ADMIN portal. `RoleContext(scope=ADMIN, orgId=null, ...)` là mô hình đúng hơn cho multi-portal. Hai cơ chế này hiện tồn tại song song — dự kiến merge `roleIds` vào `RoleContext` khi refactor User domain.

---

## Value Objects

### UserPassword
- Lưu dạng hashed (bcrypt).
- Null nếu user chỉ dùng social login.
- Domain không bao giờ expose raw password.

---

## Entity: SocialConnection

Liên kết giữa User và tài khoản mạng xã hội bên ngoài. Một User có thể có nhiều SocialConnection (mỗi provider một record).

```
SocialConnection
├── provider       (String — GOOGLE, GITHUB, ...)
├── socialId       (String — ID từ IdP)
├── email          (String — email từ IdP)
└── connectedAt    (Instant)
```

> Chưa có nghiệp vụ unlink SocialConnection.

---

## Entity: RoleContext

Tập hợp các role user có trong một scope+org cụ thể. Child entity của User — không phải aggregate root.

```
RoleContext
├── id          (Long — generated by persistence)
├── scope       (Scope — ADMIN | OPERATOR | TENANT | RESIDENT)
├── orgId       (String — null khi scope=ADMIN, định danh org khi scope khác)
└── roleIds     (Set<RoleId> — mutable, quản lý qua User aggregate)
```

**Ví dụ:**
```
{ scope=ADMIN,    orgId=null,        roleIds=[SUPER_ADMIN] }
{ scope=OPERATOR, orgId="building-1", roleIds=[PROPERTY_MANAGER] }
{ scope=RESIDENT, orgId="unit-10b",  roleIds=[RESIDENT] }
```

**Invariants:**
- `(scope, orgId)` là unique per user — không cho phép hai RoleContext cùng scope+org.
- `scope` và `orgId` immutable sau khi tạo — chỉ `roleIds` có thể thay đổi.
- Mutation `addRole()` / `removeRole()` là package-private — chỉ User aggregate được gọi.

## Trạng thái (UserStatus)

```
PENDING  → user mới tạo, chưa verify email hoặc chưa complete profile
ACTIVE   → đang hoạt động bình thường
LOCKED   → bị khoá (do admin hoặc do oauth2 service notify sau N lần login fail)
```

### Transition rules

```
PENDING → ACTIVE    [PLANNED — chưa implement, tạm thời tạo xong là ACTIVE]
ACTIVE  → LOCKED    — admin gọi lock API, hoặc oauth2 service notify sau N lần login fail
LOCKED  → ACTIVE    — admin gọi unlock API
*       → DELETED   [PLANNED — chưa có use case]
```

> Hiện tại PENDING → ACTIVE chưa implement — user tạo ra sẽ ở trạng thái `ACTIVE`.

## Hành vi

### Profile & Credential

| Hành vi          | Điều kiện       | Mô tả                                                     |
|------------------|-----------------|-----------------------------------------------------------|
| `updateProfile`  | status = ACTIVE | Cập nhật username (1 lần), fullName, phoneNumber          |
| `changePassword` | status = ACTIVE | Đổi password, verify password hiện tại trước              |
| `connectSocial`  | —               | Thêm SocialConnection; idempotent nếu đã tồn tại          |

### Status

| Hành vi   | Điều kiện       | Mô tả                                                      |
|-----------|-----------------|------------------------------------------------------------|
| `lock`    | status = ACTIVE | Khoá tài khoản — gọi bởi admin hoặc oauth2 service notify  |
| `unlock`  | status = LOCKED | Mở khoá tài khoản — gọi bởi admin                          |

### Flat Role Assignment

| Hành vi        | Mô tả                                          |
|----------------|------------------------------------------------|
| `assignRoles`  | Gán thêm roles (additive, không replace)       |
| `removeRole`   | Gỡ một role                                    |

### RoleContext Management

| Hành vi                 | Guard                          | Mô tả                                           |
|-------------------------|--------------------------------|-------------------------------------------------|
| `addRoleContext`        | (scope, orgId) chưa tồn tại   | Tạo RoleContext mới với initial role set         |
| `removeRoleContext`     | (scope, orgId) phải tồn tại   | Xoá toàn bộ RoleContext                          |
| `assignRoleToContext`   | (scope, orgId) phải tồn tại   | Thêm một role vào RoleContext                   |
| `removeRoleFromContext` | (scope, orgId) phải tồn tại   | Gỡ một role khỏi RoleContext                    |

## Invariants
- `email` là immutable sau khi tạo.
- `username` chỉ chứa `[a-z0-9._-]`, không có space.
- `username` immutable sau khi tạo đối với local registration.
- `username` cho đổi một lần duy nhất đối với social login — sau khi đổi thì immutable.
- `password` null khi user chỉ dùng social login.
- Ít nhất một trong `username`, `email`, `phoneNumber` phải có giá trị.
- Mỗi `(scope, orgId)` chỉ được tồn tại một `RoleContext` per user.
- Chỉ User aggregate được mutate `RoleContext.roleIds`.

## Events

| Event                      | Trigger                              | Handler                                      |
|----------------------------|--------------------------------------|----------------------------------------------|
| `UserCreatedEvent`         | User được tạo lần đầu (bất kể nguồn) | Gửi email welcome/verify [PLANNED]           |
| `UserLockedEvent`          | User bị khoá                         | Notify oauth2 service revoke active sessions |
| `UserUnlockedEvent`        | User được mở khoá                    | Không cần handler hiện tại                   |
| `UserPasswordChangedEvent` | Password được đổi                    | [NOTE] Nên revoke sessions — chưa implement  |
| `SocialConnectedEvent`     | SocialConnection được thêm vào User  | Không cần handler hiện tại                   |
