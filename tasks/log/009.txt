Prompt Đợt 1: Backend Development & API Contract (Phần 1 & 2)

  Mục tiêu: Xây dựng các endpoint truy vấn dữ liệu toàn hệ thống và logic Revoke dành cho Admin.

  > Subject: Thực hiện Task 009 (Phần 1) — Monitoring Backend & API
  >
  > Vai trò: Bạn là Senior Backend Engineer thực hiện các tính năng giám sát an ninh cho oauth2-service.
  >
  > Tài liệu căn cứ:
  > 1. Quy trình Task: @tasks/009_admin_global_monitoring.md (Thực hiện Task 1 và Task 2).
  > 2. Phân tích nghiệp vụ: @docs/business_analysis/admin_security_management.md.
  > 3. Use Case chi tiết: @services/oauth2/docs/use-cases/UC-012_global_login_history.md và
  UC-013_global_active_sessions.md.
  >
  > Nhiệm vụ cụ thể:
  >
  > 1. Triển khai UC-012 (Global Login Log):
  >    - Tạo slice: application.login_activity.admin_query.
  >    - Viết Handler hỗ trợ filter: ip, result, username và Pageable.
  >    - Lưu ý: Bỏ qua việc lọc theo userId của caller (check ROLE_ADMIN thay thế). Trả về đầy đủ định
  danh người dùng.
  >
  > 2. Triển khai UC-013 (Active Sessions & Admin Revoke):
  >    - Query: Slice application.session.admin_query. Lấy các bản ghi ACTIVE (Join Device để lấy tên
  thiết bị).
  >    - Action Revoke: Slice application.session.admin_revoke.
  >        - Handler gọi SessionTerminationService để hủy token.
  >        - Audit: Phải lưu vết adminId thực hiện (ghi log hệ thống hoặc DB log tùy thiết kế hiện tại).
  >        - Phát SessionRevokedEvent.
  >
  > 3. Presentation Layer:
  >    - Bổ sung các endpoint admin tương ứng vào LoginActivityController và SessionController.
  >    - Sử dụng @PreAuthorize("hasAuthority('ROLE_ADMIN')").
  >
  > 4. API Contract: Cập nhật @docs/api/oauth2-sessions-history.yaml khớp với logic vừa viết.
  >
  > Yêu cầu Handoff (Bắt buộc):
  > Sau khi xong code, hãy cung cấp một "FRONTEND CONTEXT BLOCK" chứa:
  > - Các Endpoint URLs mới.
  > - Typescript Interfaces cho: GlobalActivityLog và ActiveSession.
  > - Danh sách các Query Parameters và Enums liên quan.
  >
  > Hãy thực hiện và liệt kê các file đã thay đổi.

      TypeScript Interfaces

      // UC-012 — Global Login Activity
      export type LoginResult = 'SUCCESS' | 'FAILED_WRONG_PASSWORD' | 'FAILED_ACCOUNT_LOCKED' |
      'FAILED_SOCIAL';
      export type LoginProvider = 'LOCAL' | 'GOOGLE';

      export interface AdminLoginActivityView {
        id: string;
        userId: string | null;
        username: string;
        result: LoginResult;
        ipAddress: string;
        userAgent: string;
        deviceId: string | null;
        deviceName: string | null;
        provider: LoginProvider;
        createdAt: string; // ISO 8601
      }

      export interface AdminLoginActivityPage {
        data: AdminLoginActivityView[];
        meta: { page: number; size: number; total: number };
      }

      export interface AdminLoginActivityFilter {
        ip?: string;
        result?: LoginResult;
        username?: string;
        page?: number;   // default 0
        size?: number;   // default 20, max 100
      }

      // UC-013 — Active Sessions
      export interface ActiveSessionView {
        sessionId: string;
        userId: string;
        username: string | null; // lấy từ login_activities, có thể null
        deviceName: string | null;
        ipAddress: string;
        createdAt: string; // ISO 8601
      }

      export interface ActiveSessionsResponse {
        data: ActiveSessionView[];
      }

      │ 02002 │ 409  │ Session đã REVOKED/EXPIRED — không còn ACTIVE │
